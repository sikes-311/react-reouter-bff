# /process-feedback - 開発プロセスへのフィードバック反映コマンド

planning・dev の実施中にユーザーから指摘・改善提案を受けた際、その内容を開発プロセス文書に反映するかを確認し、承認された場合に該当ファイルを更新します。

**Claude が自律的に呼び出す**: `/planning` または `/dev` の実施中にユーザーから成果物への指摘・改善提案・修正依頼を受けたと判断した時点で、このスキルを呼び出してください。実装コードへの修正依頼ではなく、**プロセスや設計方針に関わる指摘**が対象です（例: Gherkin の書き方・レビュー観点・エージェントへの指示方法・アーキテクチャ判断）。

---

## このコマンドが実行するプロセス

```
Step 1: フィードバック内容の整理
Step 2: 反映候補ファイルの特定
Step 3: ユーザー確認
Step 4: ファイル更新
```

---

## Step 1: フィードバック内容の整理

直前の会話から受けたフィードバックを以下の形式で整理してください。

```
## フィードバック内容

**問題**: {何が問題だったか}
**改善方針**: {どう変えるべきか}
**具体例**:
  - 悪い例: {現状}
  - 良い例: {あるべき姿}
```

---

## Step 2: 反映候補ファイルの特定

フィードバックの内容に応じて、以下から該当する文書を特定してください。

| 文書 | パス | 反映すべき内容 |
|---|---|---|
| アーキテクチャ | `ARCHITECTURE.md` | ディレクトリ構成・レイヤー設計・認証フロー |
| 開発ルール | `DEVELOPMENT_RULES.md` | コーディング規約・テスト方針・命名規則・実装パターン |
| 計画プロセス | `.claude/commands/planning.md` | Gherkin の書き方・設計方針・シナリオ設計ルール |
| 実装プロセス | `.claude/commands/dev.md` | エージェントへの指示・レビュー観点・ループ制御 |
| BFF エージェント | `.claude/agents/bff-agent.md` | Controller / UseCase / Gateway の実装パターン |
| フロントエージェント | `.claude/agents/frontend-agent.md` | loader / action / useFetcher のパターン |
| BFF テストエージェント | `.claude/agents/bff-test-agent.md` | Unit / Slice / Integration テストパターン |
| フロントテストエージェント | `.claude/agents/frontend-test-agent.md` | Integration テストパターン |
| E2E エージェント | `.claude/agents/e2e-agent.md` | Playwright + Docker Compose のパターン |
| コードレビューエージェント | `.claude/agents/code-review-agent.md` | レビュー観点・チェックリスト |
| セキュリティエージェント | `.claude/agents/security-review-agent.md` | セキュリティチェックリスト |

複数の文書にまたがる場合はすべて列挙してください。

---

## Step 3: ユーザー確認

以下のフォーマットでユーザーに提示してください。

```
## 開発プロセスへのフィードバック

今回の指摘を今後の開発プロセスに反映しますか？

**フィードバック内容**
{Step 1 で整理した内容}

**反映候補ファイル**
| ファイル | 反映箇所 | 変更内容 |
|---|---|---|
| {パス} | {セクション名や行} | {追記・修正内容} |

[yes / 一部だけ（詳細を記載） / skip]
```

- **yes** → Step 4 へ進む
- **一部だけ** → 指定された範囲のみ Step 4 で更新する
- **skip** → 更新せず終了

---

## Step 4: ファイル更新

承認された変更を Edit ツールで反映してください。

### 更新の原則

- **追記優先**: 既存の記述を削除せず、ルールや例として追記する
- **具体例を含める**: 良い例・悪い例をセットで記載する
- **理由を添える**: なぜそのルールが必要かを一言添える

更新完了後、以下を出力してください。

```
✅ 開発プロセスを更新しました

更新ファイル:
  - {パス}: {変更概要}

引き続き {現在の作業} に戻ります。
```
